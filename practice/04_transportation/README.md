# 04. 運輸問題（Transportation Problem）

> 難度：中級｜變數型態：連續｜類型：純 LP（網路流結構）

## 情境

一家飲料廠有 3 個倉庫，要供貨給 4 家門市。
每個倉庫的庫存有限、每家門市的需求固定，倉庫到門市的每箱運費都不同。
問題：怎麼配送運費最低？

這題對應 PuLP 官方 Case Study「A Transportation Problem」
（https://coin-or.github.io/pulp/CaseStudies/a_transportation_problem.html），
寫完可以跟官方版本對照。

（數字為示範用虛構資料）

**運費（元/箱）**：

| | 台北門市 | 台中門市 | 高雄門市 | 花蓮門市 |
|---|---|---|---|---|
| 桃園倉 | 20 | 45 | 80 | 90 |
| 彰化倉 | 50 | 15 | 40 | 95 |
| 台南倉 | 75 | 35 | 15 | 110 |

**供給**：桃園倉 300、彰化倉 400、台南倉 300（合計 1000）
**需求**：台北 250、台中 300、高雄 350、花蓮 50（合計 950）

供給 > 需求（不平衡問題），所以供給端用 ≤、需求端用 =。

## 數學模型

**決策變數**：x_{w,s} ≥ 0，倉庫 w 送到門市 s 的箱數。

**目標式**：min Σ_{w,s} cost_{w,s} · x_{w,s}

**限制式**：

```
每個倉庫 w：Σ_s x_{w,s} ≤ supply[w]   （出貨不能超過庫存）
每家門市 s：Σ_w x_{w,s} = demand[s]   （需求要剛好滿足）
```

## 學習重點

1. 二維決策變數：`LpVariable.dicts("ship", (warehouses, stores), ...)`
2. 供需不平衡的處理（≤ vs =）；若需求 > 供給則問題無解（Infeasible），
   實務上會加虛擬倉庫（短缺懲罰）處理
3. 這類問題的限制矩陣有特殊結構（totally unimodular），
   LP 鬆弛解天生就是整數——所以連續變數解出來也會是整數

## 延伸練習

- 把台北需求改成 500（總需求 > 總供給），觀察 Infeasible，
  再加一個「虛擬倉庫」（運費 = 每箱短缺罰金 200 元）讓模型永遠有解
- 加上「桃園倉到花蓮沒有航線」（把該格運費設成極大值，或乾脆不建那個變數）
- 每條路線加上單線運量上限 150 箱，看成本增加多少
