# 01. 生產組合問題（Product Mix）

> 難度：入門｜變數型態：連續｜類型：純 LP

## 情境

一家手搖飲店每天要決定三種飲品各做幾杯，讓當日利潤最大。
每種飲品的利潤與資源消耗不同，而店裡的工時、珍珠、茶葉、鮮奶每天都有限。

（數字為示範用虛構資料）

| 飲品 | 利潤(元/杯) | 工時(分/杯) | 珍珠(克/杯) | 茶葉(克/杯) | 鮮奶(毫升/杯) |
|------|------------|------------|------------|------------|--------------|
| 珍珠奶茶 | 35 | 3 | 50 | 8 | 150 |
| 水果茶   | 45 | 5 | 0  | 10 | 0   |
| 鮮奶拿鐵 | 40 | 4 | 0  | 6  | 200 |

每日資源上限：工時 960 分鐘、珍珠 6000 克、茶葉 1500 克、鮮奶 30000 毫升。

## 數學模型

**決策變數**：x₁, x₂, x₃ ≥ 0，分別為三種飲品的杯數（先允許連續值）。

**目標式**：max 35x₁ + 45x₂ + 40x₃

**限制式**：

```
工時：3x₁ + 5x₂ + 4x₃ ≤ 960
珍珠：50x₁          ≤ 6000
茶葉：8x₁ + 10x₂ + 6x₃ ≤ 1500
鮮奶：150x₁ + 200x₃ ≤ 30000
```

## 學習重點

1. `LpProblem` / `LpVariable` / `lpSum` 的基本用法
2. **Shadow price（影子價格）**：LP 求解後每條限制式的 `pi` 屬性，
   代表「該資源多一單位，利潤能多多少」——用來判斷哪個資源最稀缺、最值得擴充。

## 延伸練習

- 把杯數改成整數變數（`cat=LpInteger`），觀察解與求解時間的變化
- 新增一種飲品，或把某項資源上限 +10%，看最佳解怎麼移動
- 加入「珍珠奶茶至少要做 50 杯」的行銷需求，觀察利潤損失多少
