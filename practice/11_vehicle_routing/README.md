# 11. 車輛路徑問題（Vehicle Routing）

> 難度：進階（本集最難）｜變數型態：0/1 + 整數｜類型：混合整數規劃（MILP）

## 情境

物流站有 2 台貨車（每台載重上限 30 單位），要服務 6 個客戶：

| 客戶 | 座標 | 需求 |
|------|------|------|
| A | (2, 4) | 10 |
| B | (5, 2) | 8 |
| C | (7, 8) | 12 |
| D | (3, 7) | 6 |
| E | (8, 3) | 9 |
| F | (6, 6) | 7 |

貨車從物流站 (0, 0) 出發、送完回站。每個客戶恰好由一台車服務一次。
目標：**總行駛距離最短**。

（座標與需求為示範用虛構資料）

## 為什麼這題難

完整的 VRP 要同時決定「誰載誰」和「以什麼順序送」，是經典的 NP-hard 問題。
這題採用實務常用的分解策略 **cluster-first, route-second**（先分群、再排路線），
把一個難題拆成兩個 PuLP 可以輕鬆處理的小模型：

### 第一階段：分群（指派模型）

**一個陷阱先講在前面**：直覺會想用「站點到客戶的往返距離 × 指派變數」當目標式，
但每個客戶都必須被指派給某台車，這個總和**不管怎麼分都是同一個常數**——
目標式對決策毫無鑑別力，solver 回傳任何一組可行解都算「最佳」。
寫完模型看到怪解，先檢查目標式是不是根本沒在區分好壞。

有效的做法（Fisher–Jaikumar 式的 seed 分群）：先挑兩個相距最遠的客戶當**種子**，
再把其他客戶指派給「離種子近」的車：

- 決策變數：a_{v,c} ∈ {0,1}：客戶 c 是否由車 v 服務
- 目標式：min Σ dist(c, seed_v) · a_{v,c}　（每個客戶靠近自己車隊的種子）
- 限制式：

```
每客戶一台車：Σ_v a_{v,c} = 1                 （對每個客戶 c）
載重上限：    Σ_c demand[c] · a_{v,c} ≤ 30    （對每台車 v）
種子綁定：    a_{v, seed_v} = 1
```

### 第二階段：排路線（每台車一個 TSP）

對每台車，在「站點 + 該車的客戶」上解**旅行推銷員問題（TSP）**。
只寫「每點進一次、出一次」是不夠的——solver 會回傳
`站→A→站` 和 `B→D→B` 這種**子迴圈（subtour）**，每點進出各一次全部滿足，
但根本不是一條完整路線。

消除子迴圈用 **MTZ（Miller–Tucker–Zemlin）公式**，給每個點一個「拜訪順序」變數：

- x_{i,j} ∈ {0,1}：是否走 i → j 這段
- u_i ∈ {1..n−1}：客戶 i 的拜訪順序（站點不用）
- 目標式：min Σ dist(i,j) · x_{i,j}
- 限制式：

```
出度：Σ_j x_{i,j} = 1                          （對每點 i）
入度：Σ_i x_{i,j} = 1                          （對每點 j）
MTZ： u_i − u_j + n·x_{i,j} ≤ n − 1            （對每對客戶 i ≠ j）
```

MTZ 的直覺：若走了 i → j（x=1），限制式變成 u_i + 1 ≤ u_j，
強迫「順序遞增」；子迴圈裡繞一圈回到自己會推出 u_i < u_i 的矛盾，所以不可能成立。
繞過站點則不受限（u 不含站點），這就是為什麼合法路線活得下來。

## 學習重點

1. **常數目標式陷阱**：目標式要能區分「好解」和「壞解」，寫完先想想它會不會是常數
2. **分解策略**：難問題拆成「指派＋排序」兩個小模型，是實務求解 VRP 的正統起手式
3. **子迴圈與 MTZ**：進出度限制的不足、順序變數如何堵住子迴圈
4. **規模意識**：MTZ 是 O(n²) 條限制式且 LP 鬆弛很弱，CBC 解十幾個點沒問題，
   幾百個點就該換工具（Google OR-Tools 的 routing solver、LKH 等啟發式）

## 延伸練習

- 把種子換成 D、E（它們相距同樣是 √41）重跑：分群和總距離會變嗎？
  體會啟發式分解**不保證全域最佳**，種子的選法本身就是一個研究題
- 改成 3 台車、每台載重 20，分群和路線怎麼變？
- 進階：不分兩階段，直接建單一模型（x_{v,i,j} 三索引變數 + 每台車各自的 MTZ），
  比較解的品質和求解時間——體會為什麼實務要用分解
- 查一下 Google OR-Tools 的 `RoutingModel`，同一題用它解一次
