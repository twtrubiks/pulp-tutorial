# 08. 數獨（Sudoku as 0/1 Programming）

> 難度：進階｜變數型態：0/1（三維）｜類型：純可行解問題（無目標式）

## 情境

數獨看起來是玩具，但它是體會 0/1 建模威力的絕佳題目，
也是 PuLP 官方 Case Study 之一
（https://coin-or.github.io/pulp/CaseStudies/a_sudoku_problem.html）。

它展示兩件前面題目沒出現過的事：

1. **沒有目標式**——我們不是在「最佳化」，只是要找一個滿足所有限制的可行解
2. **三維 0/1 變數**——「第 r 列第 c 行是否填數字 v」，
   用 729 個 0/1 變數把「填什麼數字」這種離散選擇線性化

## 數學模型

**決策變數**：x_{v,r,c} ∈ {0,1}，第 r 列第 c 行是否填入數字 v（v, r, c ∈ 1..9）。

**目標式**：無（隨便給常數 0 即可，solver 只找可行解）。

**限制式**：

```
每格恰好一個數字：   Σ_v x_{v,r,c} = 1     （對每個格子 r,c）
每列每個數字恰好一次：Σ_c x_{v,r,c} = 1     （對每個 v, r）
每行每個數字恰好一次：Σ_r x_{v,r,c} = 1     （對每個 v, c）
每宮每個數字恰好一次：Σ_{(r,c)∈宮b} x_{v,r,c} = 1（對每個 v, 宮 b）
題目給的提示數字：    x_{v,r,c} = 1
```

## 學習重點

1. 「這格填 1～9 其中一個」這種**離散選擇**，標準手法就是升維成 0/1 變數
   ——同樣的技巧用在排課（哪節課排哪個時段）、排程（工單排哪台機器）
2. 無目標式的問題：`prob += 0` 即可，關注點全在可行性
3. 若題目多解，solver 只回傳其中一個；想列出所有解，
   可以把「排除剛剛那組解」的限制式加回去再解一次（官方 case study 有示範）

## 延伸練習

- 把幾個提示數字拿掉，觀察是否出現多解（用上面第 3 點的技巧數數看有幾個解）
- 挑戰「對角線數獨」變體：兩條對角線也要 1～9 不重複（各加 9 條限制式）
- 用同樣的三維 0/1 手法，把 03 排班題改寫成「員工 × 日期 × 班別」版本
