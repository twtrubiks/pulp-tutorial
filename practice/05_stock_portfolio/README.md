# 05. 選股組合問題（Stock Portfolio Selection）

> 難度：中級｜變數型態：0/1 + 整數｜類型：混合整數規劃（MILP）

## 情境

你有 100 萬台幣，想從觀察名單裡挑股票建立投資組合，目標是**預期年報酬金額最大**，
但要遵守自己的紀律：

- 只能用整數「張」買（1 張 = 1000 股）
- 單一個股投入金額 ≤ 資金的 25%（不重壓）
- 單一產業投入金額 ≤ 資金的 40%（不押單一族群）
- 至少持有 5 檔（強迫分散）
- 有選的股票至少投入 8 萬（避免持股太零碎）

> **重要**：以下股票、股價、預期報酬率**全部是虛構的示範資料**，
> 不是任何真實公司，預期報酬率在現實中也無法這樣拍腦袋給定。
> 這題的價值在建模技巧；實際使用時，把資料換成你自己研究出來的估計值。

| 股票 | 產業 | 股價(元) | 預期年報酬率 |
|------|------|---------|-------------|
| 晶鑫半導體 | 半導體 | 580 | 12% |
| 宏達電子   | 電子   | 95  | 9%  |
| 富安金控   | 金融   | 28  | 6%  |
| 泰昌銀行   | 金融   | 45  | 7%  |
| 藍海航運   | 航運   | 120 | 15% |
| 綠能風電   | 綠能   | 65  | 10% |
| 康健生技   | 生技   | 210 | 11% |

## 數學模型

**決策變數**：

- nᵢ ∈ {0, 1, 2, ...}：股票 i 買幾張
- yᵢ ∈ {0, 1}：是否持有股票 i（**輔助變數**，用來表達「有選才有下限」這種邏輯）

令 costᵢ = 股價ᵢ × 1000（一張的成本）。

**目標式**：max Σ 報酬率ᵢ · costᵢ · nᵢ

**限制式**：

```
預算：      Σ costᵢ·nᵢ ≤ 1,000,000
單檔上限：  costᵢ·nᵢ ≤ 250,000                （對每檔 i）
產業上限：  Σ_{i∈產業g} costᵢ·nᵢ ≤ 400,000    （對每個產業 g）
至少五檔：  Σ yᵢ ≥ 5

連動限制（linking constraints，本題核心）：
  nᵢ ≤ Mᵢ·yᵢ            （沒選就不能買；Mᵢ = 單檔上限內最多能買的張數）
  costᵢ·nᵢ ≥ 80,000·yᵢ  （有選就至少投入 8 萬）
```

## 學習重點

1. **0/1 變數 + 連動限制**是 MILP 的靈魂：
   「如果選了才…」這種邏輯條件，靠 `n ≤ M·y` 和 `n ≥ m·y` 這對不等式線性化
2. big-M 的 M 不要亂設大數字——這裡用「單檔上限最多能買幾張」當自然上界，
   M 越緊，solver 越快
3. PuLP 只能做線性模型（官方定義："linear and mixed integer programming modeler"，
   https://coin-or.github.io/pulp/ ）——所以 Markowitz 的變異數（二次式）做不了；
   想加風險維度要用可線性化的風險指標（如 MAD）或改用其他工具（cvxpy 等）

## 觀察

跑出來會發現晶鑫半導體（12%）和雲端資服（13%）雖然報酬率高卻沒被選——
因為它們**一張的成本（58 萬、34 萬）就超過單檔上限 25 萬**，整張制下根本買不了。
模型忠實反映了規則，這不是 bug，是紀律和現實的取捨被攤開來了。

## 延伸練習

- 允許零股：把張數變數改成「股數」（整數、以 1 股為單位），
  高價股就買得起了，看組合怎麼變
- 把「至少 5 檔」改成「恰好 5 檔」（Σy = 5），比較報酬差異
- 加入「金融股至少一檔」的條件
- 加一欄「風險分數」，限制加權平均風險分數 ≤ 某門檻（這是線性的，做得到）
- 把資料換成你從 Goodinfo 整理的真實股價與自估報酬，跑一份自己的組合
